# 03. Workspace and Authoring

The Azure ML workspace is the control center for assets, runs, and deployments.

## Workspace Taxonomy

- Compute instances and clusters.
- Experiments and jobs.
- Data assets and models.
- Endpoints and monitoring views.

## Authoring Options in 101

- Notebooks for interactive development.
- AutoML for guided model selection.
- Designer for visual pipelines.
- Prompt flow as an introduction to LLM-driven workflows.

## Beginner Guidance

Start with notebooks and simple tracked experiments. Then use AutoML and Designer to understand alternative workflows before introducing advanced MLOps concerns.
